  • 精進
    • ABC 60 D: 4 番目の制約があまりにきな臭い．
      • dp[i][j][k] = (1 から i までの物を j 個選んで重さが k 以下となる物の選び方のうち，選んだ物の価値の総和が最大となるもののその総和) とおく．
      • 通常このテーブルは大き過ぎるが，全ての物の重さから w_1 を減じたものを考へると， k は高々 4N なので，このテーブルの大きさは O(N^3) であり，更新処理も O(N^4) でギリギリ間に合ふ．
        • ちなみに愚直に全ての i, j に対して dp[i][j] のサイズを 4N+1 にすると， Python では実行時間制限に引つかかる（一敗）．代はりに dp[i][j] のサイズを 4j+1 にすれば間に合つた．
      • 遷移式はすぐ立つたが，実装にかなり時間がかかつてしまつた．実装力の無さ．

    • ABC 175 D:
      • もし初めにマスが選べず，必ず 1 から始めるなら次のやうにすれば O(N) で解ける．
        • マス 1 から始めて，次にマス 1 に戻つてくるまでのスコアの列を s_1, ..., s_M とする（ s_1 = C_{P_1} で， s_M は s_{M-1} + C_1 である）．
        • K が十分大きいとき，具体的には K >= M が成り立つとき，
          • s_M が負ならこのループは回せば回すほど損をする．つまり s が最も大きくなるところで止めるのが最適であり，得られるスコアの最大値は max {s_1, ..., s_M} である．
          • 逆に s_M が正ならこのループは回せば回すほど得をする．このループは最大 K // M 回回せる．従つて得られるスコアの最大値は，
            • K が M で割り切れるとき， K//M * s_M と (K//M-1) * s_M + max {s_1, ..., s_M} の大きい方である．前者は後者に含まれるので，後者だけ考へれば良い．
            • K が M で割り切れないとき， K//M * s_M + max {s_1, ..., s_{K%M}} である．
        • K < M のときは，このループは一度も完全には回せない．従つて max {s_1, ..., s_K} が得られるスコアの最大値である．
      • 最初のマスを 2, 3, ..., N として同様に得られるスコアの最大値を得ていき，最後に全ての最大値を取ればそれが答へであり，つまり O(N^2) で解けた．
